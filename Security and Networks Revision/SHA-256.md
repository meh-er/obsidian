#SaN 

- It produces a **256-bit** hash value from a **512-bit block** input, meaning it is extremely hard to find two different inputs that produce the same hash (**collision resistance**) and to reverse engineer the original input from its hash (**pre-image resistance**)
- It is vulnerable to **length extension attacks** (generating a hash for a longer message) but can be mitigated using **HMAC** (Hash-based Message Authentication Code), and to **brute force attacks**, but with so many inputs this is still pretty impractical.

Process:
1. **Input Prep** (pad data to ensure it fits into fixed size chunks
2. **Initial Setup** (initial constant hash values)
3. **Processing Data in blocks** (split into 512 blocks processed in 64 rounds of operations)
	1. **Break into smaller chunks** (each 512 bit block divided into 16 smaller 32 bit blocks)
	2. **Expand chunks** (the 16 chunks are expanded into 64 using logical operations)
4. **Compression Function** (Processes each 512-bit block in 64 rounds)
	1. **Mixing and transforming** (for each round, algorithm uses a combo of AND, OR, XOR, modular addition and bit shifts)
5. **Producing final hash** (Final hash values are combined to produce 256-bit hash)
